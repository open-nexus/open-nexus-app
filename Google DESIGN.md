# OpenYASKE デザインガイドライン (DESIGN.md)

本ドキュメントは、時間割アプリ「OpenYASKE」のビジュアルデザイン、レイアウト、コンポーネント一貫性における基本原則と実装規約を定義します。開発時や新機能追加時は、必ず本ガイドラインに準拠してください。

---

## 1. コア・デザインコンセプト

OpenYASKE は、**フラットでプレミアムな「タイル・カード」デザイン**をベースとしています。
基本的には、**ダッシュボード（ホーム）のモジュラーウィジェット**および**「その他」画面（Launchpad）のグリッドリンク**のシンプルかつ洗減された佇まいをアプリ全体のビジュアルの「基準」とします。

設定（Settings）、ライセンス（More/About）、データ管理（Data Management）のような「長大なフォームや複雑な設定項目」を持つ画面は、この原則から外れて煩雑になりやすいため、以下の個別ルールに厳格に従ってデザインの統一を保つ必要があります。

---

## 2. ビジュアルの基本原則

### 🎨 2.1 配色 (Colors)
- **カラーパレットの完全準拠**: UI要素、科目、TODO、出席記録などのすべての色は、テーマパックシステム（`themeStore.pack`）で提供されるパレット変数（CSS変数）からのみ取得してください。
- **HEX値のハードコーディング禁止**: マークアップ内の `style="background: #3b82f6"` や Tailwind の `bg-[#3b82f6]` など、カラーコードの直接記述は厳禁です（ESLint ルール `theme-tokens/no-raw-hex` でチェックされます）。
- **テーマ連動**: 明暗モード（Light/Dark）やカスタムテーマパックに対応するため、常に CSS 変数（例: `var(--color-primary-500)`, `var(--color-surface-card)`）を介して参照してください。

### 📐 2.2 角丸 (Border Radius)
角丸はアプリ全体で定義された2つの主要トークンのみを使用し、アドホックな値（`rounded-xl`, `rounded-pill` 等）の使用を避けてください。
1. **`rounded-chip` (`var(--radius-chip)`)**:
   - 対象: ボタン、フォーム入力フィールド（input, select, textarea）、タブ選択肢（セグメントコントロール）、バッジ、チップ。
   - 意図: 手に馴染む適度な丸みを持たせた操作パーツ。
2. **`rounded-card` (`var(--radius-card)`)**:
   - 対象: ウィジェット、配置カード、モーダル、ボトムシート、設定セクションなどの大きなコンテナ。
   - 意図: 情報のグルーピングを示す独立したカード枠。

### 🌑 2.3 シャドウ (Shadows / 影の完全排除)
- **フラットデザインの徹底**: アプリケーション全体で、カードやウィジェット、セクションコンテナに対する静的な影（Tailwind の `shadow-sm`, `shadow` など）は原則**使用しません**。
- **境界線による表現**: 要素の区切りや階層の表現は、境界線（`border-[var(--color-surface-border)]`）および背景色の明暗（`bg-[var(--color-surface-card)]` と `bg-[var(--color-surface-muted)]` の使い分け）のみで行ってください。
- **例外**: 浮き出し効果が必要な「一時的オーバーレイコンポーネント」（モーダルダイアログの `shadow-[var(--shadow-modal)]`、ボトムシートの `shadow-[var(--shadow-sheet)]`）のみ、テーマで規定された浮き出し用の影の使用を許可します。

### 🎬 2.4 アニメーションとインタラクション (Animations)
- **不要なスケールアニメーションの排除**: リスト項目のホバーやクリック時の `active:scale-[0.99]` や `hover:scale-[1.005]` などの不均一なアニメーションは、UI の静的プレミアム感と一貫性を崩すため原則**廃止**とします。
- **ホバーアクションの統一**: ボタンやカードリンクのインタラクションは、変形ではなく背景色のスムーズな遷移（`transition-colors hover:bg-[var(--color-surface-muted)]`）または透明度（`opacity`）の変化に統一します。

## 3. コンポーネントおよびレイアウト仕様

### 📝 3.1 フォームとドロップダウン
- **TextField の使用義務**: テキスト入力、テキストエリアなどの通常のフォーム入力は、標準の `TextField.svelte` を使用してください。
- **セレクトボックス（プルダウン）の統一**: アプリ内のすべてのプルダウンは、枠線付きの丸角チップ（`rounded-chip border border-[var(--color-surface-border)]`）、右側に `ChevronDown` アイコンを絶対配置したスタイルに統一します。生テキストの右寄せセレクトや `▼` 文字の直書き、異なるアイコンの使用は禁止します。
  - **設定行内のプルダウン**: 行の右側にコンパクトに配置するため、`h-8 text-xs font-bold pl-3 pr-8`、右端に `ChevronDown (size=12px)` の構成で実装します。

### ⚙️ 3.2 設定画面 (Settings)
- **インライン・フィルタリング検索**: 設定検索は、ドロップダウンを浮かせる方式ではなく、検索 TextField に文字を入力した瞬間に非該当セクション（`SettingsSection`）自体を非表示（`hidden` クラス適用）にする「インライン・フィルタリング」として実装します。これによりオーバーレイバリアによる操作阻害を防ぎます。
- **設定行のデザイン**: 設定セクション内の行は `border-[var(--color-surface-border)]` による下線で均一に区切り、ラベルと操作スイッチ（`Switch` や `TextField`）のアライメント（配置）を揃えてください。

### 🔄 3.3 並び替え UI (Sort / Reorder)
- **直感的なドラッグ＆ドロップ**: 機能配置などの並び替えリストでは、左端に `GripVertical` アイコンを配し、HTML5 Drag and Drop API を用いたドラッグ移動をサポートします。
- **補助機能**: タッチ操作対応として右端にスッキリとしたデザインの `ChevronUp` / `ChevronDown` アイコンボタンを配置し、ハイブリッドで直感的な操作が可能な設計とします。

## 4. タイポグラフィと見出しルール (Typography & Headings)

本アプリ内での文字サイズおよび見出しの構成は、UIの階層関係を表現するために厳格に標準サイズに揃えます。アドホックな文字サイズ（`text-[13px]` 等）を直書きしないでください。

### 🏷️ 4.1 文字サイズ一覧 (Font Sizes)
- **`text-3xl font-black` / `text-2xl font-extrabold`**:
  - 用途: GPA数値、合計単位数、出席率などの「大きな統計情報数値」。
- **`text-base font-bold`**:
  - 用途: 画面タイトル（`PageHeader`内の`h1`）およびモーダルやボトムシートのヘッダータイトル。
- **`text-sm font-bold`**:
  - 用途: ウィジェットのタイトル、カード見出し、および各項目のプライマリな項目名。
- **`text-sm` (regular/medium)**:
  - 用途: 標準の本文、入力フォーム内の値、設定リスト項目の選択肢。
- **`text-xs` (regular/medium/semibold)**:
  - 用途: 補助テキスト、フォーム入力フィールドのラベル、サブタイトル、補足説明。
- **`text-[10px]` または `text-[11px]`**:
  - 用途: キャプション、タイムスタンプ、ライセンスクレジットなどのごく小さな補助情報。

### 📌 4.2 見出し構造 (Header Structure)
- **ページ見出し**: 1ページにつき `h1` は `PageHeader` 内の1つのみとします。
- **カテゴリ区切り**: 設定項目やフォーム一覧の大きな区切りには、標準の `SectionHeader` コンポーネントを使用してください。
  - **配色の統一**: `SectionHeader` の配色はすべてデフォルト（`variant="default"`、プライマリ色背景＋白文字）に統一し、`variant="light"` などの混在を禁止します。これにより画面上での見出しの役割と見た目を一貫させます。
- **カード内見出し**: ウィジェットやカード、詳細モーダル内の見出しは、`h3` または `h4` を使用し、文字サイズは `text-sm font-bold` または `text-base font-bold` を適用して階層を整えてください。

### ⚙️ 4.3 設定画面のタイポグラフィと行アライメント (Settings Row Specifications)
設定画面やデータ管理画面などのリスト行は、フォントサイズや上下余白の揺れを排除するために以下の仕様に完全統一します：
- **メインラベル（項目名）**: `text-sm font-medium text-[var(--color-nav-active)]` に統一。プレーンな `text-sm` や `font-bold` の使用を禁止します。
- **サブテキスト（説明、値）**: `text-xs text-[var(--color-nav-inactive)]` に統一。
- **行パディング**: `px-4 py-3.5` に統一し、リスト全体のアライメントを揃えます。
- **セレクトボックス行**: カード内に巨大な入力枠（TextField）を配置するのを避け、スイッチ行と同じく左右に並ぶ「右側インラインセレクト行（`flex justify-between items-center`）」として実装します。

## 5. アイコンの使用ルール (Icons)

アイコンはビジュアルのノイズとならないよう、機能の意味を直感的に補完する場合にのみ配置します。

### 📏 5.1 アイコンサイズ (Icon Sizes)
- **`18px`**: ナビゲーションバーのタブアイコン、ウィジェットのヘッダーアイコン、および主要機能カードのシンボルアイコン。
- **`16px`**: テキスト入力欄（`TextField`）の leading / trailing アイコン、およびボタン（`Button`）内のインラインアイコン。
- **`14px`**: 設定行のオプションアイコン、カード内の詳細メタデータアイコン（日時・マップピン等）、ドラッグハンドル（`GripVertical`）等のシステム操作系アイコン。
- **`11px`〜`12px`**: バッジ内、または補足説明内の非常に小さなインラインアイコン（外部リンクの `ExternalLink` 等）。

### 🎨 5.2 アイコン配色 (Icon Colors)
- **デフォルト・補助**: `text-[var(--color-nav-inactive)]` を使用し、コンテンツを視覚的に邪魔しないようにします。
- **アクティブ・強調**: プライマリテーマ色（`text-[var(--color-primary-500)]` や `text-[var(--color-primary-600)]`）を使用し、機能ごとの特色と結びつけます。
- カラーコードの直接指定（例: `text-blue-500` や `#666`）は禁止されています。
